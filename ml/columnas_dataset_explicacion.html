Explicación de cada columna del dataset NVD para el modelo CyberRisk

Columna
Rol
Que hace y para que sirve
CVE_ID
CVE-2024-1234
identificador
El nombre unico de cada vulnerabilidad en el mundo. Como un numero de cedula para cada fallo. No entra al modelo — se usa para mostrar el enlace al detalle oficial y en el reporte PDF de la PYME.
PUBLISHED_DATE
2024-03-15
identificador
Fecha en que el fallo fue publicado oficialmente. Se usa en la UI para filtrar "amenazas del ultimo año" y ordenar por las mas recientes.
MODIFIED_DATE
2024-06-01
identificador
Fecha de ultima modificacion del CVE. Se usa en actualizaciones incrementales: solo descargamos CVEs con esta fecha posterior a la ultima extraccion, ahorrando horas de descarga.
CVE_YEAR
2024
feature ML
Año extraido del CVE_ID. Le dice al modelo que tan antiguo es el fallo. Un CVE del 2024 sin parche es mucho mas peligroso que uno del 2015 que ya tiene solucion.
HAS_KEV
True / False
target binario
La columna mas importante del proyecto. True = hackers reales estan explotando este fallo activamente HOY. Si la PYME tiene en su stack un software con HAS_KEV=True, se dispara alerta roja inmediata sin importar el score.
DESC_EN
"A buffer overflow in Apache..."
texto NLP
Descripcion tecnica del fallo en ingles. El modelo la convierte en numeros con TF-IDF para aprender patrones semanticos. Tambien se traduce al español para mostrarle al empresario en lenguaje simple que paso.
SCORE_BASE
9.8 / 4.3 / 7.1
target numerico
Puntaje de gravedad del 0 al 10. Es el numero grande del dashboard: "Tu empresa tiene un riesgo de 8.2/10". El modelo aprende a predecir este numero para CVEs sin score oficial.
SEVERITY
CRITICAL / HIGH / MEDIUM / LOW
target clase
Etiqueta del nivel de riesgo en 4 categorias. Es el semaforo de colores de la UI: rojo=CRITICAL, naranja=HIGH, amarillo=MEDIUM, verde=LOW. Mucho mas entendible que un numero para el empresario.
VEC_ATAQUE
NETWORK / LOCAL / PHYSICAL
feature ML
Por donde entra el ataque. NETWORK = cualquiera desde internet puede atacar. LOCAL = solo si ya esta dentro del equipo. PHYSICAL = necesita acceso fisico a la oficina. La UI lo explica en lenguaje simple.
COMPLEJIDAD
LOW / HIGH
feature ML
Que tan dificil es ejecutar el ataque. LOW = cualquier persona con un script puede hacerlo. HIGH = requiere un hacker experto con condiciones especificas. Ayuda a priorizar que parchear primero.
PRIVILEGIOS
NONE / LOW / HIGH
feature ML
Si el atacante necesita una cuenta primero. NONE = ataque desde afuera sin cuenta. HIGH = solo si ya es administrador. Importante para detectar amenazas internas de empleados.
INTERACCION
NONE / REQUIRED
feature ML
Si el empleado debe cometer un error para que funcione. REQUIRED = alguien debe hacer clic en un enlace o abrir un archivo. NONE = el ataque es automatico sin que nadie haga nada.
ALCANCE
UNCHANGED / CHANGED
feature ML
Si el ataque puede saltar a otros equipos. CHANGED = el virus puede infectar toda la red de la oficina. Muy critico para PYMEs con todo en la misma red. La recomendacion incluye segmentacion de red.
IMP_CONFID
NONE / LOW / HIGH
feature ML
Si el ataque roba informacion. HIGH = pueden copiar todos los datos de clientes, contraseñas, facturas. Relevante para cumplimiento de habeas data en Colombia (Ley 1581).
IMP_INTEGR
NONE / LOW / HIGH
feature ML
Si el ataque modifica informacion. HIGH = pueden alterar precios, facturas, registros contables. Critico para PYMEs del sector retail o financiero donde la integridad de datos es esencial.
IMP_DISPON
NONE / LOW / HIGH
feature ML
Si el ataque apaga el sistema. HIGH = ransomware, el negocio se detiene completamente. Es la consecuencia mas entendible para cualquier empresario: "no puedo trabajar".
FACILIDAD
0.0 — 3.9
feature ML
Puntaje numerico de que tan automatico es el ataque. Alta correlacion con HAS_KEV: cuando es 3.9 (maximo) casi siempre hay exploits automatizados en circulacion. Feature muy poderosa para predecir explotacion activa.
CWE_ID
CWE-89 / CWE-79
texto NLP
Tipo de error que causo el fallo. CWE-89 = inyeccion SQL, CWE-79 = XSS, CWE-119 = desbordamiento de memoria. Agrupa CVEs por familia de error para dar recomendaciones generales por patron.
CPE_VENDOR
microsoft / apache / google
feature busqueda
Lo que el usuario selecciona en la UI: "Microsoft", "Google", "Apache". El sistema filtra todos los CVEs de ese fabricante. Sin esta columna no es posible personalizar el analisis por empresa.
CPE_PRODUCT
windows_10 / chrome / mysql
feature busqueda
El producto especifico que usa la empresa. Junto con CPE_VENDOR permite filtrar exactamente: "quiero ver solo CVEs de Windows 10, no de Windows Server". Es la clave del analisis personalizado.
PATCH_URL
https://microsoft.com/...
solo UI
No entra al modelo. Es el enlace al parche oficial del fabricante. En la plataforma aparece como boton "Descargar parche" en el reporte de la PYME. La accion mas concreta que puede tomar el empresario.
